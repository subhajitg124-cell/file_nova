# FileNova Platform

An all-in-one secure file manipulation platform for PDFs, images, office documents, and video.

## Run & Operate

- **Frontend Dev Server**: `pnpm --filter @workspace/file-nova run dev` (Runs on port `21533` with fallback)
- **API Server Dev**: `pnpm --filter @workspace/api-server run dev` (Runs on port `8081` with automatic port collision resolver)
- **Database Migrations**:
  - `pnpm --filter @workspace/db run generate` — Generate Drizzle migration files based on schema changes.
  - `pnpm --filter @workspace/db run migrate` — Execute pending SQL migration files against the target database.
- **Type Checking**: `pnpm run typecheck` — Runs strict typecheck checks across the entire monorepo workspace.
- **Build Production**: `pnpm run build` — Automatically runs `orval` codegen prebuild, typechecks, and outputs production bundles.
- **Run Tests**: `pnpm run test` — Executes the built-in Node.js test runner for validation.
- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where Things Live

- `artifacts/file-nova/` — React + Vite frontend (Vite configuration, components, styling).
- `artifacts/api-server/` — Express backend server handling file uploads, processing routes, timeouts, and CORS limits.
- `lib/db/` — Database migration configuration, schema models, and pg pool connection.
- `lib/api-spec/` — OpenAPI specification and Orval configuration for clients/zod generation.
- `lib/api-client-react/` — React Query API hooks generated by Orval.
- `lib/api-zod/` — Zod schemas generated by Orval.
- `scripts/` — CI/CD helper scripts (like git `post-merge` hook with timeout controls).

## Architecture Decisions & Security Guards

1. **Port Collision Guard**: Both the React dev server and the Express API server implement runtime port conflict checks. If the preferred port is occupied, the servers scan and bind to the next free port instead of crashing.
2. **CORS Security**: Wildcard origins (`*`) are disallowed. Allowed CORS origins are dynamically validated against a regex allowing only `localhost` and Replit workspace URLs (`*.replit.app`, `*.repl.co`).
3. **HTTP Protections**: The API server employs `helmet` for security headers and `express-rate-limit` to restrict brute-force upload operations (limited to 100 uploads per 15 minutes per IP).
4. **File Size Limits**: Multer limits uploads to a maximum of 50MB per file to prevent Denial of Service (DoS) attacks.
5. **Server-Side Validation**: Uploaded files undergo Zod mime-type validation on the server to prevent malicious extensions from being processed.
6. **Temp File Lifecycle**: Uploaded files are written to a temp folder. Immediately upon download completion or task failure, the API server purges the associated input and output files. A background garbage collection worker sweeps and deletes expired files older than 1 hour.
7. **SPA 404 Routing**: The Express server implements a wildcard route fallback (`*`) serving `index.html` in production for all non-API paths, preventing direct URL navigation 404s.
8. **Optimized Animations**: Framer Motion uses a lazy-loading setup (`LazyMotion` + `domAnimation`) in `App.tsx` and loads components using `m` instead of `motion` to reduce bundle size by 95KB+.
